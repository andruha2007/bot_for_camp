# Бот для лагеря

Бот для ВКонтакте, автоматизирующий регистрацию участников, управление мероприятиями, мини-курсами, балансом и ярмаркой.

## Структура проекта

| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа — инициализация и запуск бота |
| `bot_logic.py` | Логика бота: обработчики команд, состояния (FSM), клавиатуры |
| `database.py` | Работа с SQLite: таблицы, регистрация, баланс, мероприятия |
| `config.py` | Загрузка конфигурации из .env |
| `vk_server.py` | Реализация LongPoll сервера VK API |
| `camp.db` | Файл базы данных SQLite |

# Инструкция по работе с ботом

## Роли

- **Участник** — зарегистрированный пользователь с персональным кодом
- **Админ** — может управлять мероприятиями, курсами, жалобами, балансом
- **Супер-админ** — полный доступ + управление ярмаркой и админами

## Процесс регистрации

### 1. Пред-регистрация (админ)
Администратор вводит список участников: `Фамилия Имя, Фамилия Имя`.  
Бот генерирует для каждого уникальный персональный код вида `a123xy`.

### 2. Вход участника
Участник нажимает **Войти** и вводит свой персональный код.  
После успешного входа код становится недействительным — второй человек не сможет войти по тому же коду.

### 3. Пригласительный код (админ)
В меню **Настройки → Добавить админа** генерируется код.  
Новый администратор нажимает **Войти** и вводит этот код. Код одноразовый.

---

## Защита от дублирования регистрации

Система гарантирует, что два участника не могут зарегистрироваться по одному уникальному номеру:

1. **Флаг использования** — код проверяется на `is_used=0`. После успешного входа код помечается как использованный.
2. **Уникальность в БД** — поле `personal_code` имеет ограничение `UNIQUE` в таблицах `participants` и `pre_registered_participants`.
3. **Привязка к VK ID** — один VK-аккаунт не может зарегистрироваться дважды (`user_id INTEGER UNIQUE`).
4. **Если участник уже вошёл**, повторная попытка входа с тем же кодом вернёт ошибку `"Код не найден"`, так как код уже использован.

## Запуск ярмарки

1. Создайте мероприятие и опубликуйте его
2. Дождитесь, пока участники зарегистрируются и сформируют команды
3. В меню **Настройки → 🏪 Запустить/Остановить ярмарку** выберите мероприятие
4. Введите бюджет для каждой команды
5. Участники нажимают **🏪 Перейти к ярмарке** в главном меню
6. Команды выставляют товары, совершают покупки
7. Администратор подтверждает или отклоняет транзакции
