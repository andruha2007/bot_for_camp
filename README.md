# Бот для лагеря (bot_for_camp)

Этот бот предназначен для управления лагерными мероприятиями, мини-курсами и участниками.

## Архитектура

Бот имеет следующую архитектуру:

### Роли
- **Суперадмин**: Главный администратор с полными правами
- **Админ**: Администратор с правами управления
- **Участник**: Обычный участник лагеря

### Функциональность для участников

1. **Регистрация**: Участники регистрируются по личному коду (формат: 1 буква, 3 цифры, 2 буквы, например `a123bc`)
2. **Баланс**: Просмотр текущего баланса и истории операций
3. **Мероприятия**: Регистрация команд на мероприятия
4. **Мини-курсы**: Регистрация на мини-курсы с выбором слота времени
5. **Информация**: Просмотр информации о доступных мини-курсах
6. **Красная кнопка**: Отправка жалоб администратору

### Функциональность для администраторов

1. **Управление участниками**: Просмотр информации об участниках
2. **Мероприятия**: Создание и управление мероприятиями
3. **Мини-курсы**: Создание, публикация и управление мини-курсами
4. **Жалобы**: Просмотр и обработка жалоб от участников
5. **Настройки**: Управление админами и настройками бота

## Установка

1. Создайте виртуальное окружение:
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # На Linux/Mac
   .venv\Scripts\activate     # На Windows
   ```

2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

3. Создайте файл `.env` в корневой директории со следующим содержимым:
   ```
   VK_BOT_TOKEN=ваш_токен_бота_vk
   VK_GROUP_ID=ваш_id_группы_vk
   INITIAL_SUPER_ADMIN=ваш_vk_user_id
   DB_PATH=camp.db
   ```

4. Запустите бота:
   ```bash
   python main.py
   ```

## Структура базы данных

Бот использует SQLite базу данных с следующими основными таблицами:

- `participants`: Информация об участниках
- `events`: Мероприятия
- `event_registrations`: Регистрации на мероприятия
- `mini_courses`: Мини-курсы
- `time_slots`: Слоты времени для мини-курсов
- `mini_course_registrations`: Регистрации на мини-курсы
- `balance_history`: История операций с балансом
- `complaints`: Жалобы от участников

## Основные команды

### Для участников:
- `/start` или "Начать" - Начать регистрацию
- "Проверить баланс" - Просмотреть текущий баланс
- "Записаться на мероприятие" - Зарегистрировать команду на мероприятие
- "Записаться на мини-курс" - Зарегистрироваться на мини-курс
- "Красная кнопка" - Отправить жалобу администратору

### Для администраторов:
- "Участники" - Управление участниками
- "Мероприятия" - Управление мероприятиями
- "Мини-курсы" - Управление мини-курсами
- "Жалобы" - Просмотр и обработка жалоб
- "Настройки" - Управление настройками бота

## Особенности реализации

1. **Генерация личных кодов**: Уникальные коды в формате `a123bc`
2. **Валидация данных**: Проверка форматов ввода участников
3. **Ограничения**: Проверка доступности мест на мини-курсах
4. **История операций**: Полная история изменений баланса
5. **Управление доступом**: Разделение прав между ролями
